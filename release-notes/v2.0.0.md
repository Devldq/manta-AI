# Manta v2.0.0

## 新功能

- **多 Agent 协作调度系统** — 全新 Agent 调度操作系统，核心理念为 "Humans steer. Agents execute. Drivers are pluggable."，支持多个 AI Agent 在人工监督下高效协作完成复杂软件工程任务

- **工作流引擎** — 支持 YAML 声明式定义工作流，内置五种步骤类型：`sequential`（顺序）、`parallel`（并行）、`human_in_loop`（人工审批）、`conditional`（条件分支）、`loop`（循环）；并行步骤使用 `Promise.allSettled` 同时启动所有分支，human_in_loop 步骤暂停执行等待人工操作；内置 `dev-standard`（标准开发流程）、`quick-fix`（热修复流程）、`plateco-frontend-dev`（14 阶段完整前端开发流程）三套预置工作流

- **任务双模式调度** — 支持轻量模式（Task → Runner → 结果，直接指定 Agent 执行）和完整模式（Task → Workflow → Runner，通过工作流引擎多步骤编排）；任务全生命周期由 6 状态状态机管理：`inbox → planning → running → done / failed / archived`

- **Agent Loop 流式引擎** — 手动 while 循环实现真流式输出，使用 Vercel AI SDK v6 的 `streamText` + `stepCountIs(1)` 强制单步执行；多级退出条件：安全步数上限、Token 预算超限、循环检测强制中断、自然结束；模块化系统 Prompt 融合 Claude Code 设计理念

- **三级循环检测器** — 检测 Agent 陷入死循环的三种模式：`general-repeat`（同一工具反复调用）、`ping-pong`（两操作交替）、`polling`（连续 poll 无进展）；三级严重度递进：warning（5 次）→ critical（8 次）→ circuit-breaker（10 次，强制中断）

- **Claude Code 风格工具集** — 内置 Bash、Read、Write、Edit、MultiEdit、Glob、Grep、WebFetch、WebSearch、TodoRead/TodoWrite 等工具；Bash 工具具备安全检查，禁止 `rm -rf /` 等危险命令；工具注册中心支持全局读写锁、延迟加载、结果自动截断

- **MCP 协议完整实现** — 支持 Local（Stdio 子进程通信）、Remote HTTP（StreamableHTTPClientTransport + Headers 认证）、Mock 三种模式；完整实现 OAuth 2.0 + PKCE 授权流程（RFC 8252 Loopback Server）；工具可见性控制支持 per-agent glob 模式过滤

- **可插拔 Runner/Driver 系统** — Runner 接口抽象（`probe()` 环境检测 + `run()` 执行任务），内置 OpenClaw Runner 通过 CLI 启动 Agent 进程；注册表模式，新增 Runner 只需实现接口并注册

- **插件系统** — 声明式注册，每个插件包含 `plugin.yaml` 声明 `id`、`name`、`runnerId`、`agentFormat`；启动时自动扫描 `plugins/` 目录加载；内置 OpenClaw（读取已注册 Agent）和 Plateco（前端需求开发全流程 Agent 集合）两套插件

- **多 LLM 提供商支持** — 支持 OpenAI、OpenAI-Compatible（DeepSeek/通义千问/Moonshot 等）、Ollama、LM Studio 四种提供商；多模型 Profile 配置，可在设置中创建和切换不同模型配置；MiMo 推理模型适配，通过自定义 fetch 实现 `reasoning_content` 字段自动注入和回传

- **60+ 设计主题系统** — 业界最大规模的产品设计 token 集合，从 Apple、Linear、Notion、Vercel、Stripe、Cursor、Tesla、Spotify 等 65 个知名产品设计规范提取；反 FOUC 主题注入，在 React 水合前写入 CSS 变量消除闪屏；支持亮暗模式全局切换

- **通知系统** — 支持 macOS 系统通知和 Webhook 通知（飞书、Slack、钉钉、Discord、Generic 五种 payload 格式）；ChannelManager 全局管理器并行发送

- **文件安全系统** — CWD 内自由访问，CWD 外需用户实时授权；授权请求 → 用户批准/拒绝 → 路径白名单持久化

- **日志系统** — LogManager 单例架构，支持 ToolCall、AgentLoop、System、Error、Performance、Custom 六种日志类型；实时订阅机制、统计报表、导出功能

- **Electron 桌面端** — 基于 Electron 41 打包，支持 GitHub Releases 自动更新

- **敏感信息检测** — 15+ 内置检测规则：API Key、Bearer Token、JWT、私钥、AWS 密钥、密码、数据库连接串、公司邮箱/域名、手机号、身份证号；Git pre-commit 钩子自动检测，可自定义规则和严重程度

## 技术栈

- **前端** — Next.js 15 + React 19 + TypeScript 5
- **样式** — Tailwind CSS 3 + CSS 变量主题系统
- **AI** — Vercel AI SDK v6 + LangChain
- **MCP** — @modelcontextprotocol/sdk v1.29
- **状态管理** — Zustand v5
- **桌面端** — Electron 41 + electron-builder + electron-updater
