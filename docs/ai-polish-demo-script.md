# AI 润色功能演示脚本

## 演示场景 1: 新建 Agent（完整流程）

### 准备工作
- 确保已配置 `.env` 中的 API Key
- 启动应用并访问 http://localhost:3000/agents

### 演示步骤

**Step 1: 打开新建弹窗**
```
1. 点击右上角「+ 新建」按钮
2. 弹窗打开，显示表单
```

**Step 2: 填写基础信息**
```
名称: code-reviewer
描述: 专业的代码审查助手，提升代码质量和团队协作效率
```

**Step 3: 使用 AI 润色助手**
```
核心职责:
- 自动化代码质量检查
- 识别潜在安全漏洞
- 提供最佳实践建议
- 确保代码符合团队规范

工作流程:
1. 接收代码变更通知
2. 运行 ESLint、Prettier、SonarQube 等工具
3. 分析代码复杂度和可维护性
4. 生成结构化审查报告
5. 提供具体改进建议和代码示例
```

**Step 4: 点击 AI 生成**
```
1. 点击「✨ AI 生成 SOUL.md」按钮
2. 按钮显示「✨ AI 生成中...」
3. 等待 2-5 秒
4. SOUL.md 编辑器自动填充生成的内容
```

**Step 5: 审阅和调整**
```
1. 滚动查看生成的内容
2. 发现 AI 生成的结构清晰、内容专业
3. 可以手动微调某些细节
```

**Step 6: 创建 Agent**
```
1. 点击「创建并注入」按钮
2. Agent 创建成功
3. 自动注入到 OpenClaw
4. 在左侧列表中看到新建的 Agent
```

## 演示场景 2: 编辑现有 Agent（优化流程）

### 准备工作
- 已有一个基础的 Manta Agent
- 内容不够完善，需要优化

### 演示步骤

**Step 1: 选择 Agent**
```
1. 在左侧「Manta」分区选择一个 Agent
2. 右侧显示详情面板
3. 当前 SOUL.md 内容比较简单
```

**Step 2: 进入编辑模式**
```
1. 点击右上角「编辑」按钮
2. 描述变为可编辑
3. SOUL.md 变为 textarea
4. 顶部出现「✨ AI 润色」链接
```

**Step 3: 展开 AI 润色助手**
```
1. 点击「✨ AI 润色」链接
2. 展开润色助手面板
3. 显示「补充职责」和「补充流程」输入框
```

**Step 4: 补充信息**
```
补充职责:
- 增加代码风格一致性检查
- 支持多语言项目审查
- 集成 CI/CD 自动化流程

补充流程:
- 与 Git hooks 集成
- 提供实时审查反馈
- 支持团队审查规则自定义
```

**Step 5: AI 优化**
```
1. 点击「✨ AI 优化 SOUL.md」按钮
2. 等待优化完成
3. 编辑器内容更新
4. 显示「✓ AI 润色完成」提示
```

**Step 6: 保存变更**
```
1. 审阅优化后的内容
2. 点击「保存」按钮
3. 显示「已保存」提示
4. 自动重新注入到 OpenClaw
```

## 演示场景 3: 错误处理（展示健壮性）

### 场景 3.1: 未配置 API Key

**演示步骤**:
```
1. 确保 .env 中没有 API Key（或重命名）
2. 尝试点击 AI 生成按钮
3. 显示友好的错误提示：
   「未配置 AI API Key（环境变量 OPENAI_API_KEY 或 AI_API_KEY）」
4. 提示用户检查配置
```

### 场景 3.2: 空输入禁用

**演示步骤**:
```
1. 打开新建 Agent 弹窗
2. 不填写任何信息
3. AI 生成按钮为禁用状态（灰色）
4. 鼠标悬停显示提示
```

## 关键亮点展示

### 亮点 1: 智能上下文理解
```
AI 会基于：
- Agent 名称
- 描述
- 职责
- 流程
- 现有 SOUL 内容

综合生成符合上下文的专业文档
```

### 亮点 2: 结构化输出
```
生成的 SOUL.md 始终包含：
- # Agent · SOUL 标题
- ## 身份 章节
- ## 职责 章节
- ## 工作流程 章节

格式规范、易读、专业
```

### 亮点 3: 渐进式优化
```
支持两种模式：
1. 从零生成：基于基础信息创建
2. 增量优化：基于现有内容改进

用户可以多次迭代优化
```

### 亮点 4: 无缝集成
```
- 与新建流程完美融合
- 与编辑流程自然衔接
- 不影响原有手动编辑能力
- 生成后可继续手动调整
```
