## Design Context

### Product
**Manta** — 软件工程智能体（Software Engineering Agent），核心能力是**并行处理多项任务**。用户可同时启动多个子任务（编码、搜索、分析、部署等），Agent 在后台并发执行，用户通过统一面板监控进展、查看结果。

### Users
- **Target audience**: 软件工程师与高级开发者，需要 AI 协作完成大规模、多线程的工程任务
- **Context**: 桌面端优先（Electron + Web），支持亮/暗双模式，以**任务看板 + 会话**为核心交互模型
- **Job to be done**: 同时追踪和管理多个 AI 子任务的进展，快速切换上下文，高效决策
- **Mental model**: 像「任务指挥官」— 分发指令，并行监控，按需介入

### Brand Personality
- **3-word**: Precise, Parallel, Commanding（精准、并行、掌控）
- **Metaphor**: Manta（蝠鲼）— 幽灵般滑行于深海，双鳍如翼，无声而精准地覆盖大片海域（并行视野）
- **Emotional goals**: 掌控感、效率感、专业信赖 — 用户感觉自己在「指挥」而非「等待」

### Aesthetic Direction
- **Visual tone**: 精密工具感 — 干净、克制、专注。像手术器械或航空仪表盘：信息密度高但无冗余，每条数据都有其位置
- **Color**: Emerald 绿（行动/确认）+ Warm Stone 暖灰中性（亮色）/ Deep Abyss 冷黑（暗色）。绿色为唯一强调色，避免多色干扰并行任务时的注意力
- **Typography**: DM Sans（正文，消除 Inter 的 AI 痕迹）+ JetBrains Mono（代码/数据，精准感）
- **Reference**: Linear（视觉层次）、Figma（多画布并行的空间感）、Raycast（瞬时交互）、Aircraft cockpit HUD（信息密度与优先级）
- **Anti-reference**: 通用 AI 生成界面（Inter 字体、青紫渐变、毛玻璃、发光点缀、hero metrics 模板、单线程聊天窗口）

### Design Principles
1. **Parallel-first**: 每个 UI 组件都应考虑「同时有 N 个实例在运行」— 紧凑但可区分，状态一目了然
2. **Content-first**: 界面服务于信息，不反向
3. **Tinted neutrals**: 绝不用纯黑/纯白/纯灰 — 所有中性色必须带 subtle 色调（chroma ≥ 0.003）
4. **Easing must be named**: 不用 CSS 默认 ease，所有过渡使用命名缓动曲线（ease-out-quart/expo）
5. **Focus for all**: 所有可交互元素必须有 focus-visible 指示器，支持 reduced-motion
6. **Space with rhythm**: 分组的紧凑（4-8px）、区间慷慨（24-48px），用间距区分并行任务的边界

### Technical Constraints
- CSS 变量驱动的 auto-theme 系统，运行时热切换
- oklch 色彩空间（感知均匀）
- next/font 加载字体，避免 FOUT
- 壁纸系统通过 ::before/::after 实现
- Tailwind CSS + 内联 style 混合策略
